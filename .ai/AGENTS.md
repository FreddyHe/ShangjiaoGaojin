# .ai 工作区索引

> 本目录是 AI Agent 的项目记忆库。每次开启新会话时，**优先阅读本文件**，再按需加载其他文件。

## 项目简介

**新闻稿智能体** — 基于 Vue 3 + FastAPI 的新闻稿自动生成系统，部署在阿里云 ECS（IP: `47.102.216.195`），通过 Nginx 提供服务。支持 JWT 认证（admin/user 角色）。

核心能力：上传素材（文件/URL） → AI 智能分类 → 选择大纲模板 → 人名匹配与冲突检测 → 流式生成新闻稿 → 支持多版本迭代修改。附带多分类知识库管理（人物、品牌、课程等）和 SAIF 官网教授数据同步。

## 文件索引与阅读策略

| 文件 | 内容 | 何时阅读 |
|------|------|----------|
| `SOUL.md` | Agent 人格、行为准则、沟通风格 | **每次会话开始时** |
| `PITFALLS.md` | 易踩坑规则，主动防御清单 | **执行任何操作前必读** |
| `ARCHITECTURE.md` | 项目架构、目录结构、核心数据流 | 需要理解项目结构时 |
| `OPERATIONS.md` | 服务启动、环境配置、常用脚本 | 需要启动/重启/部署服务时 |
| `CONVENTIONS.md` | Git 规范、代码风格、命名约定 | 写代码或提交代码时 |
| `LESSONS.md` | 已解决问题的归档 | 遇到难以定位的 bug 时翻阅 |
| `CURRENT_TASK.md` | 当前任务上下文 | 从新窗口恢复工作时 |

## 工作流程

```
新会话启动
  ├─→ 读 AGENTS.md（本文件）
  ├─→ 读 SOUL.md（确认行为准则）
  ├─→ 读 PITFALLS.md（加载防御规则）
  │
  ├─ 如果是写代码 ──→ 读 CONVENTIONS.md
  ├─ 如果是改架构 ──→ 读 ARCHITECTURE.md
  ├─ 如果是运维操作 ──→ 读 OPERATIONS.md
  ├─ 如果是排查 bug ──→ 读 LESSONS.md
  └─ 如果是接续上次工作 ──→ 读 CURRENT_TASK.md
```

## 维护规则

- **PITFALLS.md**：发现新的反复性坑时立即追加，保持条目简短（每条不超过 3 行）
- **LESSONS.md**：每次解决重大 bug 后归档，格式为「日期 + 问题 + 根因 + 方案」
- **ARCHITECTURE.md** 如果调整了项目结构，包括增删改查文件，请更新
- **CURRENT_TASK.md**：切换任务前更新，完成任务后清空
- 所有文件使用中文撰写，代码和路径保持英文原样
